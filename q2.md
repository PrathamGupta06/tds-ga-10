## Why Marp for product docs?
- Single markdown source in git; reviewable and diffable
- Export to PDF/PPTX/HTML via `marp` CLI for stakeholders
- Directives keep styling close to content without complex build
- Speaker notes and fragments support live walkthroughs

## Maintainability checklist
- Store `q2.md` with PR reviews for every change
- Keep snippets minimal; link to repos instead of duplicating code
- Use `<!-- notes: ... -->` for presenter context without clutter
- Favor directives (`_class`, `_color`, `backgroundColor`) over inline HTML

---

## Converting formats quickly
```bash
# Render to HTML
marp q2.md --html --output docs.html

# Render to PDF with chrome (default)
marp q2.md --pdf --allow-local-files

# Render to PowerPoint
marp q2.md --pptx
```
